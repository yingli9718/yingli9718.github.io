Airbnb: optimize listing booking and price

Background and Motivation

Airbnb provides an online marketplace for short-term home and apartment rentals. The mission of the platform is to create a triple win scenario to help 1) the hosts rent out their properties and maximum their earnings, 2) the travelers find the best listing that suites their needs, and 3) Airbnb maximize the revenues.

From the listing hosts' perspective, they need to 1) determine the best price for their Airbnb, 2) describe the listing in an optimized way to maximize their booking rates, and thus their revenue.

Goal of the Project

The aim of this project is to analyze the publicly available data from Airbnb to develop two solutions/tools:

  1. Booking Optimizer (predict booking score for a listing)

    Goal: to provide insights and recommendations for the listing host to take actions to optimize his books and thus revenue.

     

    Value: 1) provide a score that helps listing hosts understand the predicted booking level of their listing based on the provided information. 2) provide a recommendation list that helps listing hosts to take actions to optimize and improve the booking.

  2. Listing Price Optimizer (to predict the “right” for a listing)

    Goal: to help the listing hosts to maximize the revenues.

     

    Value: 1) provide a recommended "best" listing price. 2) provide a recommended top improvements/action items of what drives the price and what can be improved to improve the price.

What's novel

  1. Unstructured text data

    leverage the unstructured text data from both the listing description and guests’ comment

  2. Booking optimization and recommendation

Part 1

The following two parts of the dataset are used for this projects:

  • Open Source Data

    The group named Inside Airbnb scrapes and compiles publicly listing data for years and for multiple cities around the world from the Airbnb website. The historical data are archived on monthly basis. For this project, I am using these historical monthly data for model training and evaluation.

  • Webscraping Data

    This GitHub repo shows how to use Selenium andBeautifulsoup4 to webscraping all listings for a given city. For this project, I am using the webscraping current listing data for future prediction.

    Two categories of data:

    1. Listing information: bed type, bathroom, location, minimum nights, maximum nights, amenities, etc. Unlike many other analysis on Airbnb, I will also leverage the un-structured listing description (text) data. The hypothesis is that the way one describes a list may have an influence on viewer's perception and thus drives book.
    2. Guest Comments: I will leverage the unstructured past guest comments data as well in the model. For example, I will assess the positivity of each of the comment by applying some text and sentiment analysis techniques and take that into the model.

Part 5

  • Model Building

    Regression, tree-based model, XGBoost, Neural network

    • Booking Opimizier

      For this project, I have to use the number of comments as a proxy of booking. However, if Airbnb likes this solution and can provide the actual booking data, the pipeline can be rerun and adjusted accordingly.

    • Listing Price Opimizier

      To build this model, the price of the listing will be the target and the listing information, description texts, as well as past guests' comment will be used as the independent/predictor data.

  • Model Validation

    • Booking Opimizier

      The booking optimizer model will be validated majorly on the rank-ordering. The model will provide a score on a scale of e.g. 1-99 on the predicted level of booking. For example, if you can a score of 90 that means your listing is on the top 10% for similar listings. This can be easily received by nontechnical stakeholders.

    • Listing Price Opimizier

      The price optimizer model will be validated majorly on level MSE and accuracy of the prediction, which is easily interpretable to nontechnical stakeholders.
